So sánh
4 Directions — Đánh giá
So sánh các tradeoff để chọn direction hoặc kết hợp elements. Câu hỏi quyết định: bạn muốn thấy results trong khi replay đang chạy không?

A — Terminal Classic Recommended

  • Results panel luôn visible — xem live update trong replay
  • Closest to Bloomberg/TradingView mental model
  • Không cần toggle hay interaction để xem results
  • Resize handle cho trader tự điều chỉnh
  • Chart bị compress ~70% — ít không gian hơn
  • Phù hợp nhất với "live scoring feeling" emotional goal

B — Focus Mode

  • Chart 100% width — maximum immersion khi replay
  • Overlay không push chart — không có layout shift
  • Results hidden trong Setup/Replay — không thấy live update
  • Overlay che một phần chart khi visible
  • Toggle interaction thêm cognitive load
  • Phù hợp nếu trader chỉ muốn xem chart, đọc results sau

C — Balanced Split

  • Tabs (Overview/Per-Trade/Export) tổ chức tốt hơn
  • Resize handle rõ ràng nhất
  • Phù hợp màn hình rộng (27" monitor)
  • Panel tabs thêm complexity — MVP không cần
  • Nhiều UI elements hơn cần thiết
  • Có thể adopt tab idea cho Phase 2

D — Bottom Panel

  • Chart full width — tốt nhất cho laptop nhỏ (13")
  • Gross/Commission/Net visible trong table — audit trail rõ
  • Completion summary overlay trên chart elegant
  • Linked selection chart ↔ table khó implement (khác axis)
  • Marker → table scroll phức tạp hơn với horizontal layout
  • Bottom panel gross/commission detail có thể adopt vào A
Recommendation: Direction A với elements từ D
Base: Direction A (Terminal Classic) — results panel luôn visible, live update trong replay, resize handle.
Borrow từ D: Per-trade table có cột Gross / Commission / Net (thay vì chỉ Net) — audit trail đầy đủ hơn.
Borrow từ D: Completion summary overlay trên chart sau replay complete — inline, không blocking.
Borrow từ C: Resize handle visual rõ ràng hơn giữa chart và panel.
